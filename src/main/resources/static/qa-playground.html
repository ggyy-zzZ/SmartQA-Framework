知识库助手 · 操作台

按下面顺序操作即可：先学知识（文档 / 文本 / CSV / 表结构），再用最上面的提问检索与回答。 问答时 MySQL 检索走的是应用配置的库（默认 assistant），与下方「动态连接」的库可以不是同一个。

  • ① 提问
  • ② 学习知识
  • ③ 固定库 · 表结构
  • ④ 动态库 · 表结构 / 沉淀方案
  • ⑤ 行数自检
  • MiniMax 配置检查

1 提问

输入自然语言问题；勾选「流式」可边看边出字。知识范围与下方学习时保持一致（一般选「企业知识库」）。

就绪

2 学习知识（写入助手库 / 向量 / 图谱）

想学习业务表里的真实数据：请用 CSV 上传（从业务库导出 CSV）或 粘贴文本。 下面「动态 MySQL」两条按钮只学习表结构元数据，不会把各表里的行数据搬进知识库。

2.2 粘贴文本学习

适合贴一小段说明、FAQ、或从别处复制的表格文字。标题会显示在知识记录里。

2.3 上传 CSV（表行数据 → 当作文本学习）

UTF-8 CSV；首行为表头请勾选「首行是表头」。行数超过服务端上限会被拒绝（见返回 JSON）。

3 固定库：导出 Schema（仅表结构）

使用应用里配置的 MySQL（默认库 assistant），从 information_schema 生成表结构说明。可选 LLM 评估、写入知识库。
